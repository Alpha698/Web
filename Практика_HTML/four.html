Основные понятия: URL

Для задания местоположения Web-страниц в World Wide Web используется URL (Uniform Resource Locator). Каждая страница имеет свой уникальный адрес.

URL был изобретен Тимом Бернерсом-Ли в 1990 году в стенах Европейского совета по ядерным исследованиям в Женеве, Швейцария. URL стал фундаментальной инновацией в Интернете. Изначально URL предназначался для обозначения мест расположения ресурсов (чаще всего файлов) во Всемирной паутине. Сейчас URL применяется для обозначения адресов почти всех ресурсов Интернета. Стандарт URL закреплён в документе RFC 1738, прежняя версия была определена в RFC 1630. Сейчас URL позиционируется как часть более общей системы идентификации ресурсов URI, сам термин URL постепенно уступает место более широкому термину URI. Стандарт URL регулируется организацией IETF и её подразделениями.

Примеры URL:

  • http://tehlib.com.ua/ - http-сервер должен найти и отправить клиенту (браузеру) стартовую страницу сайта "Библиотека технической литературы";
  • http://tehlib.com.ua/matematika.htm - http-сервер должен найти и отправить клиенту (браузеру) файл matematika.htm.

URL включает:

  1. Метод доступа к ресурсу, т. е. протокол доступа (http-доступ к Web-странице в сети с использованием соответствующего протокола). Это наиболее часто используемый метод доступа к какому-либо HTML-документу в сети.
  2. Cетевой адрес ресурса.
  3. Полный путь к файлу на сервере.
